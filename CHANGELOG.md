# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-06-25

0.1.0 was the first alpha release.

### Added

- **Core retrieval:**
  - Added a modality-agnostic intermediate representation built around `Document`, `Unit`, `Reference`, and `StructureGraph`, with a modality-blind retrieval pipeline that indexed, fused, graph-expanded, and packed structured corpora into a `ContextBundle` plus an auditable `Receipt`.
  - Added the shipped adapters for specs and prose: `SpecAdapter` for OpenAPI and JSON-Schema sources, and `ProseAdapter` for Markdown and reStructuredText sources.
  - Added the byte-exact T0/T1 kernel path with SQLite FTS5/BM25F indexing, RRF plus relative-score fusion, bounded graph expansion, deterministic T1 reference closure, and the efficiency packer.
  - Added the opt-in T2 local vector lane for prose and natural-language retrieval behind the `[embed]` extra.
  - Added the stateless persistence model: each `index` and `query` call built its in-memory index and structure graph per run and discarded them afterward.
- **Surfaces:**
  - Added the Python library API as the source-of-truth surface for indexing in-memory IR or routing files and directories through adapters.
  - Added the `omnex` CLI with `index` and `query` commands for deterministic local retrieval.
  - Added the `omnex-mcp` FastMCP stdio server with `index` and `query` tools that returned the same T0 bundle and receipt as the library and CLI.
- **Adoption and operations:**
  - Added `omnex install-client` for six MCP clients: Claude Code, Codex, Cursor, OpenCode, Pi, and oh-my-pi.
  - Added local usage metrics as an off-by-default, local-only ledger, with CLI controls for enablement, summaries, export, deletion, and trace opt-in.
  - Added `omnex doctor` diagnostics for MCP registration, usage-metrics state, installed extras, adapter sanity, and stateless persistence reporting.
- **Integrations:**
  - Added the LangChain retriever integration as `omnex.integrations.langchain.OmnexRetriever` behind the `[langchain]` extra.
  - Added the LlamaIndex retriever integration as `omnex.integrations.llamaindex.OmnexLlamaRetriever` behind the `[llamaindex]` extra.
- **Packaging and benchmarks:**
  - Added slim and full local-build Docker images, plus a CI workflow that built both images and smoke-ran the slim image.
  - Added the labeled spec and prose benchmark families, with checked-in results artifacts generated by `omnex-bench`.

0.1.0 was not yet published to PyPI; installs used the git URL instead, for example `uv tool install "omnex @ git+https://github.com/Mathews-Tom/omnex"`.
